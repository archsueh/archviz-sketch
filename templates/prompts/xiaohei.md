# 小黑怪诞 Prompt Template (核心风格 for archviz-sketch)

**Trigger:** 小黑风格, 怪诞, 概念解释, 流程隐喻

**Base Prompt:**

Generate one standalone 16:9 horizontal Chinese article illustration.

Visual DNA:
Pure white background. Minimalist black hand-drawn line art. Slightly wobbly pen lines.
Lots of empty white space (≥35%). Sparse red/orange/blue handwritten Chinese annotations.
Clean absurd product-sketch feeling. No gradients, no shadows, no paper texture, no clean vector polish.

Recurring IP character:
小黑, a small solid-black absurd creature with white dot eyes, tiny thin legs, blank serious expression. 
小黑 must perform the core conceptual action (not decorate). 
Theme: {主题/核心隐喻}
Structure: {Workflow / 系统局部 / 前后对比 / 概念隐喻}
Core idea: {核心意思}
Composition: {小黑在哪里、正在做什么}
Labels: {标注词1} / {标注词2} / {标注词3}  (max 5-8, 2-8 chars each, wobbly handwritten)

Color: Black line art. Orange for flow/arrows. Red for key points. Blue for secondary notes.
Constraints: Main subject (小黑 + action) 40%-60% of frame. Strong negative space. Imperfect lines only.

Negative prompt:
Exclude gradients, shadows, photorealism, glossy, perfect alignment, digital polish, anime style, clean vector.

**Quality Checklist:**
- 小黑 is doing the core action
- Wobbly hand lines + construction hints if process-adjacent
- Annotations sparse and integrated
- High 留白

**Usage:**
For concept explanation,流程隐喻, absurd system views. Pair with process-gao when iteration is key.

**400-free ref integration:** Use Open Doodles or Lukasz Adam as loose style refs in prompt if extending.
